# VoiceCanvas 设计文档 — 问题追踪与技术决策

## 项目核心目标

用户通过语音实现**无感绘画操作**和**矢量图绘制能力**：
- 语音输入 → AI 理解 → 画布执行，端到端延迟 < 3 秒
- 支持自然语言描述生成矢量图形（花、树、爱心等）
- 支持语音编辑已有图形（改颜色、移动、删除、调整大小）
- 视障用户可通过 TTS 反馈了解画布状态

## 二、路由策略（Planner 层）

Planner 采用**三层路由**决定使用哪组工具：

```
第0层：强硬拦截（关键词命中直接返回）
  ├─ 知名图标关键词（VS Code/微信/Chrome…）→ icon 组
  └─ 复杂生物关键词（猫/狗/人/风景…）→ image_gen 组
      （但如果同时包含编辑/删除意图词则跳过拦截）

第1层：关键词匹配
  └─ 扫描 TOOL_GROUPS 的 keywords，匹配后按 exclusive 规则裁剪

第2层：LLM 路由
  └─ 关键词未命中时，调用小模型判断意图

兜底：默认走 create 组
```

工具组互斥关系：
- `icon` 排斥 `create`/`vector`/`vector_gen`/`image_gen`
- `delete` 排斥 `create`/`edit`/`vector`/`vector_gen`
- `vector` 排斥 `create`
- `vector_gen` 排斥 `create`/`vector`
- `image_gen` 排斥 `create`/`vector_gen`
- `edit` 与 `create` 同时匹配时，若含编辑意图词则移除 `create`

### 3.5 视障辅助（完全未实现）

### 3.6 其他缺失

| 功能         | 说明                                                    |
| ------------ | ------------------------------------------------------- |
| 打断机制     | 用户说话时无法打断 AI 回复                              |
| 颜色模糊匹配 | "深红"→标准色卡映射，依赖 LLM 理解无确定性映射          |

| 多指令拆解   | "画一个太阳，然后移到左边"依赖 LLM 多步调用，无编排引擎 |

## 四、后续优化方向

### 1. 添加 Skill（新能力扩展）
- 海龟绘图 Skill：落笔/抬笔/步数/角度，实现自由路径绘制
- 自由画笔 Skill：手绘路径，支持画笔粗细和颜色
- 相对位移 Skill："向右移动50像素"→自动计算绝对坐标
- 距离感知 Skill："离边界多远？"→返回像素距离描述
- 预览确认 Skill：画前高亮闪烁，支持语音"确认"/"取消"

### 2. 添加工具（补齐工具链）
- `pen_draw`：自由画笔工具，支持路径绘制
- `turtle_move`：海龟绘图工具，支持前进/转向/落笔抬笔
- `relative_move`：相对位移工具，基于当前位置偏移
- `get_distance`：距离查询工具，返回到边界/其他图形的距离
- `preview_shape`：预览工具，高亮显示待创建图形

### 3. 优化提示词（提升理解准确率）
- 精简 System Prompt，控制在 100 行以内
- 增加颜色模糊映射表（"深红"→#8B0000，"天蓝"→#87CEEB）
- 增加缺失参数追问逻辑（"画个圆"→追问颜色和位置）
- 优化多指令拆解（"画太阳然后移到左边"→分步执行）
- 优化路由策略：矢量意图优先于图片生成

### 4. 优化架构（提升稳定性与体验）
- TTS 语音反馈接入，操作后自动播报结果
- ASR 语音识别接入，支持持续聆听 + 唤醒词
- VAD 语音活动检测，自动判断说话开始/结束
- PROCESSING 状态命令队列，防止指令丢失
- redo() 完整实现
- 后端维护画布对象列表，减少前端→后端上下文传输
- 矢量图生成转为可交互对象（非纯图片）
- 清理 fabric-engine.js 死代码
