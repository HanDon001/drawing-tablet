## 一、项目概述

### 1.1 项目简介

VoiceCanvas 是一款**绝对零物理交互**（无鼠标、无键盘、无触屏）的纯语音控制绘图工具，专为肢体不便和视力障碍用户设计。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 🎤 纯语音控制 | 通过自然语言指令完成所有绘图操作 |
| ♿ 无障碍设计 | TTS 实时播报，视障用户友好 |
| 🤖 AI 驱动 | 理解模糊、复杂的语音指令 |
| 🎨 对象化绘图 | 支持图形创建、编辑、删除完整生命周期 |
| 💾 本地持久化 | 防抖自动保存，刷新不丢失 |
| 🔗 全链路追踪 | X-Request-ID 请求追踪 |

### 1.3 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 控制台前端 | HTML + Tailwind CSS + Canvas API | - |
| 后端框架 | FastAPI | ^0.136.3 |
| 日志系统 | Loguru | ^0.7.3 |
| 配置管理 | Pydantic Settings | ^2.13.1 |
| 语音识别 | 小米 MiMo ASR (mimo-v2.5-asr) | API |
| 语音合成 | 小米 MiMo TTS (mimo-v2.5-tts-voiceclone) | API |
| HTTP 客户端 | httpx | ^0.28.1 |

```javascript
function processCommand(text, uiType, uiValue) {
    // 1. 快通道检测
    const fastCmd = detectFastCommand(text);
    if (fastCmd) {
        executeFastCommand(fastCmd);
        return;
    }
    
    // 2. 意图解析
    const result = parseIntent(text);
    
    // 3. UI 参数覆盖
    if (uiType === 'fill') result.params.newColor = uiValue;
    if (uiType === 'size') result.params.newSize = uiValue;
    // ...
    
    // 4. 执行动作
    executeAction(result);
}
```

#### 3.2.5 Canvas 渲染引擎

```javascript
function renderAll() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    
    state.objects.forEach(obj => {
        const px = POSITION_MAP[obj.position]?.x || 0.5;
        const py = POSITION_MAP[obj.position]?.y || 0.5;
        const cx = canvas.width * px;
        const cy = canvas.height * py;
        const sz = 30 * (obj.size === 'small' ? 1 : obj.size === 'large' ? 3 : 2);
        
        // 设置样式
        ctx.globalAlpha = obj.opacity || 1;
        ctx.fillStyle = obj.color;
        
        // 绘制形状
        switch(obj.shape) {
            case 'circle':
                ctx.beginPath();
                ctx.arc(cx, cy, sz, 0, Math.PI * 2);
                ctx.fill();
                break;
            case 'rectangle':
                ctx.fillRect(cx - sz, cy - sz * 0.8, sz * 2, sz * 1.6);
                break;
            // ...
        }
    });
}
```

## 八、关键设计决策

### 8.1 为什么使用纯 HTML 而不是 Vue？

**原因**：
1. 控制台需要快速迭代
2. 减少构建复杂度
3. 直接在浏览器中打开即可测试
4. Canvas 操作更直观

**保留的 Vue 代码**：
- `web/src/` 目录保留了 Vue 组件
- 可用于后续重构或扩展

### 8.2 为什么使用小米 MiMo API？

**原因**：
1. 高质量的中文语音识别
2. 支持语音克隆（TTS）
3. 流式响应，延迟低
4. API 兼容 OpenAI 格式

### 8.3 为什么使用本地意图解析？

**原因**：
1. 当前阶段需要快速验证 MVP
2. 避免 LLM API 的网络延迟
3. 简化部署和配置

**未来计划**：
- 集成通义千问 LLM，支持更复杂的指令理解

## 十一、已知问题与限制

### 11.1 已知问题

1. **LLM 未集成**：当前使用简单正则解析，不支持复杂指令
2. **中文编码**：Windows 命令行中文显示乱码
3. **端口占用**：需要先杀死占用端口的进程

### 11.2 功能限制

1. 不支持自由画笔（海龟绘图）
2. 不支持复杂相对距离感知
3. 不支持多对象层级关系
