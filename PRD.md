# 纯语音绘图工具需求规格说明书 (PRD)

## 1. 项目概述

| 字段 | 内容 |
|------|------|
| 项目名称 | VoiceCanvas (语音画布) |
| 项目定位 | 一款绝对零物理交互（无鼠标、无键盘、无触屏）的纯语音控制绘图工具 |
| 核心价值 | 通过自然语言处理与语音识别技术，打破物理操作壁垒，让肢体不便（如高位截瘫、手部震颤）和视力障碍的用户，能够仅凭语音完成艺术表达、信息图表制作与简单设计工作 |

## 2. 用户特征与核心痛点分析

| 用户群体 | 核心痛点 | 针对性设计策略 |
|----------|----------|----------------|
| 肢体不便者 (高位截瘫/震颤) | 无法移动光标，无法精准点击，传统绘图软件完全不可用 | 抛弃传统"模拟鼠标"逻辑，采用绝对坐标系、网格系统和对象化绘图，降低精准度依赖 |
| 视力障碍者 | 看不见画布内容、不知道光标位置、无法确认绘制结果 | 引入TTS (语音合成) 实时播报机制，提供画布状态描述、光标位置朗读、对象属性反馈 |

## 3. 功能需求详细说明

### 3.1 语音交互引擎 (核心底座)

- **持续聆听与唤醒**：应用启动后进入持续监听状态，无需手动点击麦克风。
- **VAD (语音活动检测)**：自动判断用户说话的开始和结束，避免长时间静音占用资源。
- **防误触/防杂音**：支持设定"唤醒词"（如："小画小画"），或通过物理噪音阈值过滤环境杂音。
- **TTS 语音反馈**：每次指令执行后，必须提供语音确认（如："已画了一个红色圆形"），视障用户的"眼睛"。

### 3.2 画布与空间定位系统 (针对无鼠标设计)

传统绘图依赖鼠标移动，语音绘图必须重构空间逻辑：

- **网格定位法**：将画布划分为虚拟网格（如 10x10），用户可说"移动到第3排第5列"。
- **九宫格定位法**：用户可说"移动到左上角"、"移动到正中心"、"右下角"。
- **相对位移法**：用户可说"向右移动50像素"、"向上移动两格"。
- **绝对坐标法**：用户可说"坐标 200 300"。

### 3.3 基础绘图指令

- **形状绘制 (对象化)**：支持"画一个[颜色] [大小]的[形状]在[位置]"。
  - 示例："在中间画一个红色的圆"、"在左上角画一个蓝色的正方形"。
- **自由绘制 (落笔/抬笔)**：模拟海龟绘图法。
  - 示例："落笔" -> "向前走100步" -> "左转90度" -> "向前走50步" -> "抬笔"。
- **线条绘制**："从当前位置画一条线到左下角"。

### 3.4 属性与编辑指令

- **颜色修改**："把刚才的圆改成绿色"、"把画笔颜色设为黄色"。
- **大小修改**："把圆放大一点"、"把正方形缩小一半"。
- **选择与删除**："选中刚才画的圆"、"删除选中的图形"、"清空画布"。
- **撤销与重做**："撤销"、"恢复"。

### 3.5 视障辅助指令 (查询与探索)

- **位置查询**："我现在在哪里？" -> TTS播报："当前光标在画布中心偏左"。
- **内容查询**："画布上有什么？" -> TTS播报："画布中心有一个红色大圆，左上角有一个蓝色小方块"。
- **距离感知**："离边界多远？" -> TTS播报："距离右边框还有200像素"。

### 3.6 系统与导出指令

- **保存/导出**："保存图片"、"导出为PNG"。
- **帮助**："我可以说什么？" -> 播报可用指令列表。

## 4. 非功能需求

### 4.1 指令理解的准确性与容错性

- **模糊匹配**：当用户说"画一个深红的圆"，系统能映射到标准色卡中的深红色。
- **同义词兼容**："撤回"="撤销"、"删除"="擦掉"="不要了"。
- **缺失参数容错**：当用户说"画个圆"（未说颜色和位置），系统应采用默认值（如黑色、当前位置），并通过TTS询问："已用黑色在当前位置画圆，需要修改颜色吗？"

### 4.2 响应延迟控制

- **ASR延迟**：语音识别延迟应控制在 500ms 以内（采用Web Speech API或端侧模型）。
- **指令执行**：从说完到画布渲染完成，总延迟应 < 1秒。
- **反馈优先**：TTS语音确认应与画布渲染同步，不能让用户在沉默中等待。

### 4.3 复杂指令的拆解与执行

系统需具备多参数提取能力。例如："在右上角画一个比刚才那个圆大一倍的黄色三角形"

1. 拆解1：定位 -> 右上角
2. 拆解2：查找历史 -> 获取"刚才那个圆"的尺寸 (假设半径50)
3. 拆解3：计算 -> 尺寸翻倍 (半径100)
4. 拆解4：属性 -> 黄色、三角形
5. 拆解5：执行并播报。

## 5. 语音指令集规范 (V1.0 计划)

| 指令类别 | 语法结构 | 示例 | 优先级 |
|----------|----------|------|--------|
| 空间定位 | 移动到 [方位/坐标/网格] | "移动到左上角" / "坐标100 200" | P0 |
| 形状绘制 | 画一个 [颜色] [大小] 的 [形状] | "画一个红色的大圆" | P0 |
| 线条绘制 | 画线到 [方位/坐标] | "画线到右下角" | P0 |
| 海龟绘图 | 落笔/抬笔/向前/向后/左转/右转 | "落笔，向前走100步" | P1 |
| 编辑修改 | 修改 [属性] 为 [值] | "把颜色改成蓝色" | P0 |
| 历史操作 | 撤销/重做/清空 | "撤销" | P0 |
| 视障辅助 | 哪里/有什么/多大 | "画布上有什么" | P0 (视障核心) |
| 系统操作 | 保存/帮助 | "保存图片" | P1 |

## 6. 交互流程示例 (视障用户画一个笑脸)

```
用户："小画小画"
系统："我在听。"

用户："在正中间画一个黄色的圆。"
系统：(渲染黄色圆) "已在画布正中间画了一个黄色圆形。"

用户："在圆的左上方画一个小黑圆。"
系统：(计算左上方相对位置，渲染眼睛) "已在黄色圆的左上方画了一个黑色小圆。"

用户："画布上有什么？"
系统："画布正中有一个黄色大圆，其左上方有一个黑色小圆。"

用户："在黄色圆的底部画一条黑色的弧线，当做嘴巴。"
系统：(渲染弧线) "已在黄色圆底部画了一条黑色弧线。"

用户："保存图片。"
系统："图片已保存至本地。"
```

## 7. 风险评估与缓解措施

| 风险点 | 描述 | 缓解措施 |
|--------|------|----------|
| 环境噪音误触发 | 持续监听易被电视、旁人说话干扰 | 采用唤醒词机制；支持语音指令"安静模式"（仅接收精确指令格式） |
| 识别错误连锁反应 | 语音识别错一个字，画错一个图，撤销很麻烦 | 每次绘图操作前增加"预览高亮"（闪烁边框），2秒后确认，或语音说"确认"后才真正落下 |
| 空间描述主观性 | "左上方"对系统是具体坐标，对用户是模糊概念 | 提供TTS位置反馈闭环，如："已移动到左上方，距离顶部20%，距离左边15%" |
